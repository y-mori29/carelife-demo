## 1. v02 で反映した要件・実装方針

以下の内容を要件定義書 v02 および本開発計画 v02 で反映し、実装に落としています。

| 項目 | 内容 |
|------|------|
| **SC-01 起動画面** | 患者氏名を **姓** と **名** の2項目で入力。担当者名も同一画面で入力。「開始する」で次へ。患者の選択リストは廃止し、入力のみとする。 |
| **録音終了** | 「録音を終える」押下時に **終了確認画面**（SC-05a）を表示。「はい、終了する」／「いいえ、続ける」で誤操作を防止。 |
| **補足入力** | 補足質問を **1画面に全質問・全入力欄を一覧表示**。ない場合は空欄のまま「次へ」で短時間に完了。 |
| **報告送信** | 補足入力後に報告を生成し、**報告確認・編集画面**（SC-11）で内容を確認・修正できるようにする。**「LINEに送信する」** ボタンを押したときに初めてLINEへ送信。送信完了画面（SC-12）で「LINEに送信しました」を表示。 |
| **途中再開** | 可能な範囲で、入力内容をブラウザの localStorage 等に保存し、再訪時に「前回の入力を再開しますか？」と案内。実装が難しい場合は未実装でも可。 |

## 2. 画面遷移（v02）

SC-01（姓・名・担当者入力）  
→ SC-04 録音開始  
→ SC-05 録音中  
→ **SC-05a 録音終了確認**  
→ SC-06 録音完了  
→ SC-07 補足開始  
→ SC-08 補足一覧（全質問1画面）  
→ SC-09 補足確認  
→ SC-10 処理中  
→ **SC-11 報告確認・編集** → 「LINEに送信する」  
→ SC-12 送信完了  

異常時: SC-ERR エラー画面
