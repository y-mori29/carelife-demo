# LINE グループ・リッチメニュー利用手順

Carelife Bot を **グループ LINE** で使う方法と、**リッチメニューのボタン**で「報告を作成」を起動する方法をまとめます。

---

## 1. 想定している使い方

- **グループ LINE** に Bot を追加し、施設内の担当者が同じトークで利用する。
- 担当者が **キーワードを送る** か **リッチメニューのボタンを押す** と、Bot が「報告作成用リンク」を返す。
- そのリンクからブラウザで報告画面を開き、録音〜報告作成〜「LINEに送信する」まで行うと、**押した人（送った人）の LINE** に報告文が届く。

---

## 2. Bot をグループに追加する

### 2.1 事前に必要なこと

- LINE Developers で **Messaging API チャネル** を作成済みであること。
- Bot を **友だちに追加** 済みであること（1対1で「報告」と送って反応する状態）。

### 2.2 手順

1. LINE で **グループを作成** する（または既存のグループを開く）。
2. グループの **設定**（右上のメニュー）を開く。
3. **メンバーを追加** を選ぶ。
4. **「友だち」一覧から、Carelife の Bot（チャネル名）を選択** して追加する。

これで、そのグループ内で Bot がメンバーとして参加し、グループで送られたメッセージが Webhook で届くようになります。

### 2.3 注意（グループ追加ができない場合）

LINE の仕様変更により、**Messaging API の Bot をグループに追加できない** 時期・アカウントがあることが報告されています。  
その場合は次のいずれかで運用してください。

- **1対1トーク** で利用する（Bot を友だち追加し、個人で「報告」と送る）。
- LINE のアップデートや LINE Developers の案内を確認し、グループ追加が可能になったら再度試す。

---

## 3. キーワードで反応させる（現在の仕様）

Bot は次の **いずれかの文言** が送られると、報告作成用リンクを返します。

| キーワード（いずれかを含む） | 例 |
|------------------------------|-----|
| 報告 | 「報告」「通院報告お願い」 |
| 通院 | 「通院の報告です」 |
| はじめる | 「はじめる」 |
| スタート | 「スタート」 |

- **1対1** でも **グループ** でも同じです。
- **グループ** の場合、メッセージを送った **その人** の LINE に、あとで「LINEに送信する」で報告文が届きます（リンクにその人の userId が含まれるため）。

### 使い方の例（グループ）

1. グループで「報告」と送る。
2. Bot が「通院報告を作成します。下のリンクからはじめてください。」とリンクを返す。
3. そのリンクをタップしてブラウザで報告画面を開く。
4. 録音〜報告作成〜「LINEに送信する」まで行う。
5. 報告文が **自分（リンクを開いた人）の LINE** に届く。

---

## 4. リッチメニューでボタンから起動する

「報告」と打たなくても、**リッチメニューのボタン** を押したときに同じ動作（報告作成リンクを返す）にすることができます。

### 4.1 リッチメニューとは

- Bot とトーク画面を開いたとき、画面下部に表示される **メニュー（画像＋ボタン）** です。
- ボタンに「通院報告を作る」などを表示し、押したら Bot がリンクを返すようにできます。

### 4.2 実装済みの動作

Carelife の line-bot は、次の **2 通り** で報告作成リンクを返します。

| 方法 | 内容 |
|------|------|
| **テキストメッセージ** | 本文が「報告」「通院」「はじめる」「スタート」のいずれかにマッチする |
| **ポストバック** | リッチメニューのボタンで `data` が `report` または `action=report` の postback が送られてきたとき |

リッチメニューでは、**ポストバック** を使うと「報告」という文字をトークに送らずに、ボタンだけで同じ処理を実行できます（トークがスッキリします）。

### 4.3 リッチメニューの作成手順（LINE Developers）

1. [LINE Developers](https://developers.line.biz/ja/) にログインし、対象の **Messaging API チャネル** を開く。
2. **「リッチメニュー」** タブを開く。
3. **「リッチメニューを作成」** で以下を設定する：
   - **サイズ**: 例）2500 x 1686（フル）または 2500 x 843（ハーフ）
   - **デフォルトメニュー**: オン推奨（常に表示）
   - **画像**: 用意したリッチメニュー用画像をアップロード
4. **「エリアを追加」** でボタン（タップ領域）を設定する。
   - **アクションタイプ**: **ポストバック**
   - **ラベル**: 例）通院報告を作る
   - **データ**: `report` または `action=report`  
     （Carelife の Bot はこの値を検知して報告リンクを返します）
   - **表示テキスト**: 空欄で OK（トークにメッセージを出さない場合）。  
     または「報告を作成」などと入れると、ボタン押下時にその文言がトークに表示されます。
5. **「公開」** して、**「デフォルト」に設定** する。

### 4.4 リッチメニュー用画像の目安

- **フル**: 2500 x 1686 ピクセル
- **ハーフ**: 2500 x 843 ピクセル  
画像の上に「エリア」を重ねてボタン位置を決めるので、デザイン上は「ここを押す」と分かるようにしておくとよいです。

## 5. まとめ

| やりたいこと | やること |
|--------------|----------|
| **グループで使う** | グループの「メンバーを追加」から Bot（Carelife）を追加する。グループで「報告」と送るか、リッチメニューのボタンを押す。 |
| **キーワードで起動** | 1対1またはグループで「報告」「通院」「はじめる」「スタート」のいずれかを送る。 |
| **リッチメニューで起動** | LINE Developers でリッチメニューを作成し、ボタンのアクションを「ポストバック」、データを `report`（または `action=report`）に設定する。 |

Bot の Webhook と `FRONTEND_URL` が正しく設定されていれば、グループ・1対1どちらでも、キーワードとリッチメニューの両方で同じように報告作成リンクが返ります。
