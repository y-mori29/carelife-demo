## 1. プロジェクト概要

- **名前**: Carelife — 介護施設向け通院報告支援システム（MVP）
- **ルート**: `carelife_202603/`
- **目的**: LINE から起動し、録音 → 文字起こし（Speech-to-Text）→ AI要約（Gemini）→ 通院報告文の確認・編集 → LINE に送信する流れを実現する

---

## 2. ここまでに完了していること

### GCP・環境
- **プロジェクト**: `yorisoi-demo`（請求有効化済み）
- **主なリソース**: Firestore（東京）、GCS バケット `yorisoi-demo-recordings`、Artifact Registry `carelife-repo`、サービスアカウント `carelife-backend-sa`
- **参照**: `docs/GCP環境構築計画_yorisoi-demo.md`

### アプリ（ローカルで動作済み）
- **バックエンド**（`backend/`）: Node.js + Express。録音アップロード受信 → ffmpeg で WAV 変換 → GCS 保存 → Speech-to-Text V2（chirp_3）→ Gemini（gemini-3.1-flash-lite-preview）で通院報告要約。レスポンスで `transcript` と `reportText` を返す。マークダウンなし・今日の日付で報告文を生成。
- **フロントエンド**（`frontend/`）: Vite + React。患者名・担当者入力 → 録音（MediaRecorder）→ 補足質問 → 報告作成 → 文字起こし・要約の表示。「LINEに送信する」は画面遷移のみ（実際の LINE 投稿は未実装）。
- **LINE Bot**（`line-bot/`）: Webhook で「報告」「はじめる」等に反応し、報告作成用フロント URL を返す。

### Cloud Run デプロイ（完了済み）
- **carelife-backend**: ソースからビルド、PORT・GCS・Gemini 用環境変数・サービスアカウント設定済み
- **carelife-frontend**: ビルド時に `VITE_API_BASE_URL` でバックエンド URL を埋め込み、静的配信
- **carelife-linebot**: `FRONTEND_URL` と LINE の Channel Secret / Access Token を環境変数で設定
- **デプロイ方法**: `scripts/deploy-cloudrun.ps1`（backend/.env から GEMINI_API_KEY と LINE_* を読み込む）
- **参照**: `docs/Cloud_Runデプロイ手順.md`

### その他
- **FFmpeg**: バックエンドの音声変換用。Windows では winget でインストール済みの想定。
- **TypeScript**: フロントで `import.meta.env` を使うため `src/vite-env.d.ts`（`/// <reference types="vite/client" />`）を追加済み。

## 5. 未実装・今後の拡張の例

- **LINE への報告文送信**: SC-11 で「LINEに送信する」を押したときに、Messaging API の Push Message で報告文を送る（送信先の userId の渡し方・LIFF 等の設計が必要）
- **Webhook 署名検証**: line-bot で LINE の Webhook 署名検証を実装するとより安全
- **Firestore 保存**: 現在はレスポンス返却のみ。encounters 等を Firestore に保存する場合はバックエンド拡張
- **Chatwork 連携**: 要件では「既存運用に沿って」のため、まずは LINE 投稿のみでよい想定

## 6. 技術メモ（引き継ぎ用）

- **バックエンド**: `backend/server.js`（PORT は環境変数、既定 8081）。finalize は `upload.single('audio')` で multipart 受信。GCS・STT・Gemini は `carelifeController.js` の `runSttAndReport`。
- **フロント**: API ベース URL は `import.meta.env.VITE_API_BASE_URL`（未設定時は `http://localhost:8081`）。Cloud Run ではビルド時に埋め込み。
- **LINE Bot**: `line-bot/server.js`。POST `/webhook` で 200 を即返し、非同期で reply。`FRONTEND_URL` を報告リンクとして返す。
- **プロジェクト ID**: `yorisoi-demo`。リージョンは `asia-northeast1`（東京）。
